COMCTL32 4.0 Ordinal-Only Exports

The version 4.0 from the original Windows 95 introduced no new functions exported only by ordinal. Indeed, it drops many (most notably the Unicode forms of those that take string input) that are exported from versions 3.50 and 3.51 and which are restored for version 4.70 and higher.

Indeed, they are also restored for a version 4.0 that is known from a distribution of Internet Explorer for Windows NT, along with one more string utility function that is otherwise not a COMCTL32 export until version 4.71.

Ordinal Function Remarks
375 StrCSpnIW 4.0 (NT) and higher;
retired in 5.82 and 6.10 from Windows Vista, and higher;
independent implementation exported from SHLWAPI version 4.70 and higher;
documented as requiring “shlwapi.dll version 4.71 or later”
