CL /Bv

This option enables a version report at startup.

Syntax

The option is /Bv only.

Behaviour

The report is written to standard error. It shows a full pathname and a version number for each known compiler module:

  • CL itself
  • the front-end C compiler (ordinarily C1.DLL)
  • the front-end C++ compiler (ordinarily C1XX.DLL)
  • the in-between C/C++ compiler or processor (ordinarily not supplied)
  • the back-end C/C++ compiler or code-generator (ordinarily C2.DLL)
  • the linker (ordinarily LINK.EXE)
  • and MSPDB70.DLL

All modules known are listed, whether or not they will be needed for the build. (For instance, the linker is listed, even if linking is suppressed by the /c option.)

The ordinary modules, as supplied in the Visual C++ package, are not necessarily the ones that CL knows to use (and which CL names in the /Bv report). The /B1, /B2, /Bl and /Bx options name replacements for C1, C2, LINK and C1XX respectively. The /B1_5 option names the processing module—a sort of C1½—which is not ordinarily supplied.
