PROPSYS 7.0 Ordinal-Only Exports

Though the PROPSYS version 7.0 from Windows Vista SP2 exports the same functions as the version 6.0 from Windows Vista SP1, the version 7.0 from Windows 7 does have new functions that are exported only by ordinal. None seem to be documented.

Ordinal Function Remarks
418 GetDefaultKindsForExtension begins in Windows 7;
reuses ordinal from GetDefaultVisibleInForExtension
423 PSGetCanonicalValue begins in Windows 7;
reuses ordinal from PSStrFormatByteSizeEx
424 PSGetValueAndPath begins in Windows 7
425 PSGetNamedValueAndPath begins in Windows 7
426 PSGetObjectIdentity begins in Windows 7
427 PSGetSemanticTypeByName begins in Windows 7
428 PSEnumerateSemanticTypes begins in Windows 7
429 PSStrFormatByteSizeEx begins in Windows 7;
previously ordinal 423
430 GetDefaultKindsForExtension begins in Windows 7
431 GetDefaultLocationKindsForExtension begins in Windows 7
432 PropVariantConcatenateAsVector begins in Windows 7
433 CreateSnippet begins in Windows 7
434 IsPropVariantValid begins in Windows 7
