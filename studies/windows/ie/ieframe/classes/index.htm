SKETCH OF HOW RESEARCH MIGHT CONTINUE AND RESULTS BE PRESENTED

IEFRAME Classes

IEFRAME exposes very many classes through its exported DllGetClassObject function. Given that they are properly listed in the registry, these are the IEFRAME classes for which anyone who knows the corresponding CLSID may obtain a class factory by calling the well-documented OLE32 function CoGetClassObject or get an instance of the class created in one step by calling the CoCreateInstance function. Put another way, these are the IEFRAME classes that would not have noncreatable in their IDL declarations.

Warning: This list has so far been extracted only from two builds, namely version 7.0 from the original Windows Vista and version 8.0 from the original Windows 7.
