IEFRAME 8.0 Ordinal-Only Exports

The column of Remarks and the usual colour-coding for documentation status await my receipt (on disc from an MSDN subscription) of a definitive SDK for Windows 7 to adopt as a formal reference edition of Microsoft’s documentation. Meanwhile, inspection of Microsoft’s online documentation suggests that all functions that are newly exported only by ordinal for version 8.0 are undocumented, as is typical practice.

Three of these functions are not exported from the IEFRAME in Internet Explorer 8. One of these had a long history as an export from SHDOCVW, which does raise the question of why it didn’t need to be moved to IEFRAME until Windows 7.
